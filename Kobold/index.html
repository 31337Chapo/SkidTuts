Initial Access — Unauthenticated RCE via MCP
CVE-2026-23520

Browsing to http://kobold.htb:3552/api/docs reveals a full Scalar API reference with no authentication required. Reading through the endpoint list, the POST /api/mcp/connect endpoint stands out immediately — it accepts a serverConfig JSON object specifying a command, args, and env to execute, with no authentication check. This is part of the broader CVE-2026-23520 attack surface in Arcane v1.13.0.

CVE-2026-23520: Arcane v1.13.0 passes the /api/mcp/connect serverConfig payload directly to the system without authentication or sanitisation. Any unauthenticated user can achieve remote code execution on the host.

Step 2 — Trigger the Reverse Shell

Navigate to https://mcp.kobold.htb, open the developer console with F12, and paste the fetch payload. The browser is already on the correct origin so no CORS restrictions apply:

PrivateBin Template LFI → Arcane Credentials
GHSA-g2j9-g8r5-rg82

PrivateBin 2.0.2 has a template-switching feature that stores the user's chosen theme in a template cookie. When templateselection = true is set in the config (which it is here — visible from the theme switcher in the top-right of the UI), the server trusts the cookie value and includes the corresponding template file without validating the path. This allows local file inclusion by setting the cookie to a path-traversal value pointing to any readable file on the server.

The theme dropdown visible on bin.kobold.htb is the tell — if template selection is enabled in the UI, templateselection = true is set in the config, and the LFI is live.

Step 1 — Confirm RCE in the Container

Privilege Escalation — Arcane Container Deployment → Root

As an Arcane admin we can deploy new containers through the dashboard. The Docker daemon runs as root, and Arcane has access to the host's Docker socket at unix:///var/run/docker.sock — visible in the sidebar. Deploying a container with the host filesystem mounted at /hostfs and running as root gives full access to the host.

Step 1 — Create a New Project

Click Projects in the left sidebar → New Project. Paste the following compose definition and click Create Project:

services:
  pwn:
    image: privatebin/nginx-fpm-alpine:2.0.2
    user: root
    entrypoint: ["/bin/sh", "-c", "sleep 3600"]
    volumes:
      - /:/hostfs
We use privatebin/nginx-fpm-alpine:2.0.2 because it's already pulled on the host — no outbound internet access needed. The sleep 3600 entrypoint keeps the container alive so we can use the interactive shell.

Step 2 — Access the Shell

Once the container is running, navigate to Containers → click pwn-pwn-1 → click the Shell tab. Arcane provides an interactive browser-based shell directly into the container:

Unintended: Docker Group Misconfiguration
Unintended

This box has a misconfiguration that allows skipping the entire PrivateBin and Arcane steps. Ben is a member of the operator group (gid 37), and the Docker socket at /var/run/docker.sock has group ownership set to docker with srw-rw---- permissions. The operator group happens to have access to the docker socket on this host — an unintended misconfiguration by the box creator.

This was confirmed as unintended by the box creator. The docker group access via the operator group was a configuration mistake — the intended path requires going through PrivateBin and Arcane to reach the privileged container deployment.
